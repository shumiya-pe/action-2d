# ゲームデザイン仕様（第1作）

2026-07-03 のヒアリングで決定した方針。技術面の設計は [DESIGN.md](./DESIGN.md) を参照。

## 決定事項

| 項目       | 決定                  | 実装への影響                                                                      |
| ---------- | --------------------- | --------------------------------------------------------------------------------- |
| 世界観     | **SF・メカ**          | 斬撃=ブレード、射撃=ビーム、飛行=ブースター等の読み替え。skin.ts を SF 調へ       |
| ゲーム構造 | **両モード併存**      | ステージクリア型（手動スクロール）＋エンドレスラン（自動スクロール）を両方磨く    |
| 被弾・ミス | **HP制（ハート3つ）** | 接触=HP-1＋無敵時間＋ノックバック。HP0 でミス。落下死は即ミスのまま               |
| バフ持続   | **時間制のまま**      | 飛行・速度・攻撃力は秒数バランスのみ調整                                          |
| 武装持続   | **弾数制へ変更**      | 爆弾・弓は残弾制（例: 爆弾5発・矢20本）。撃ち切りで標準武装に戻る。HUD に残弾表示 |

## 未決定の設計項目（次回以降に確認）

1. **エンドレスランのスコア設計**: 距離ベースか、撃破・収集ベースか、複合か
2. **ボス戦の有無**: ステージクリア型の節目にボスを置くか
3. **難易度カーブ**: エンドレスランの速度・敵密度の上昇ルール
4. **HP回復手段**: 回復アイテムを置くか、ステージクリアで全回復のみか
5. **セーブ・進行管理**: ステージ解放状況やハイスコアの保存（現状スコープ外）
6. **操作の拡張**: 溜め攻撃・下入力（しゃがみ/急降下）などを足すか
7. **ストーリー演出**: ステージ間の会話・カットシーンの有無と分量
